LOOPFORGE - ARCHIVOS CON PREFIJO REDIS

Estos archivos hacen que Loopforge use la misma database de Upstash/Redis sin chocar con Ardent.

Cambio principal:
Todas las keys de Redis ahora empiezan con:

loopforge:

Ejemplos:
- loopforge:session:...
- loopforge:user:...
- loopforge:players

Archivos incluidos para reemplazar en tu repo:

1) api/_lib/redis.js
2) api/auth/callback.js
3) api/leaderboard.js

Cómo usar:
1. Descomprimí el ZIP.
2. Copiá la carpeta api encima de la carpeta api de tu proyecto Loopforge.
3. Cuando Windows pregunte si querés reemplazar archivos, poné que sí.
4. Subí los cambios a GitHub:

   git add .
   git commit -m "Prefix Loopforge Redis keys"
   git push

5. En Vercel, asegurate de conectar la database Upstash con prefix:

   UPSTASH_REDIS

6. Variables esperadas en Vercel:

   UPSTASH_REDIS_REST_URL
   UPSTASH_REDIS_REST_TOKEN

7. Hacé Redeploy en Vercel.

No hace falta tocar Ardent.
